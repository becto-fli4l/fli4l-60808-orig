07.03.2002 jw5
    - Added weak declaration for external symbols.

    If a packet references an externally defined variable, it may
    happen that the package defining this variable isn't present. This
    would trigger an error message in mkfli4l which expects to find
    all variables referenced by opt/package.txt.

    To handle this situation package developers can add a
    "weak"-declaration for external variables. It has the following
    format:

    opt/package.txt:

            weak        variable    -

    This introduces the variable to mkfli4l if its not already present
    and will set it to 'undefined'. If its necessary we may use the
    third field to introduce default values. The infrastructure is
    already there.
