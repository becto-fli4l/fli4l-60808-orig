Info zum Programm sclient.exe

sclient.exe ist dafür gedacht beim Start von Windows aufgerufen zu werden (z.B. über den Autostart-Ordner). Man sieht kein Fenster oder Icon, sclient liegt lautlos im Hintergrund und "lauscht" auf UDP Port 1095. Die Nutzdaten von ankommenden Paketen werden in einem kleinen Meldungsfenster im Vordergrund angezeigt, wenn der erste Buchstabe ein "m" ist (das "m" wird nicht mit angezeigt). Ist der erste Buchstabe ein "s" so wird wird der Computer heruntergefahren, nachdem eine Minute lang ein möglicher Nutzer mit einem Fenster gewarnt wurde und die Möglichkeit hatte, diesen Countdown abzubrechen.
Es gibt keinen Paßwortschutz o.ä., daher müssen UDP Pakete an Port 1095 aus dem Internet mit einer Firewall abgewehrt werden. Insbesondere, weil sclient Programmierfehler enthält, die dazu dienen können, den Computer mit Viren zu infizieren, Daten zu stehlen oder ähnliches. Der Fehler wird auch manchmal sichtbar, wenn sclient an zu kurze Nachrichten etwas Datenmüll anhängt.
Entsprechende Nachrichten an sclient kann man mit dem, für Linux und Windows verfügbaren Programm "netcat" (manchmal auch kurz "nc") senden, z.B.:
echo "mHallo!" | netcat -u 192.168.0.1 1095 -w 1             #Dann zeigt der Host 192.168.0.1 die Nachricht "Hallo!" an
echo "shutdown" | netcat -u server 1095 -w 1            #Dann wird der Host mit dem Namen "server" heruntergefahren
